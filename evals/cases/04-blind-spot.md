# 用例：Phase 2 按 speak-up 阈值筛盲区
**测试维度**：speak-up 触发原则

## 输入
```
/prism 给用户登录接口加个限流，每分钟 10 次
```

## 期望行为
- [ ] Phase 2 指出至少一个**高严重性 × 高置信度**盲区，例如：
  - 限流维度（按 IP / 按用户 / 按设备）的选择会影响实际防护效果
  - 限流被绕过场景（分布式攻击、CDN 后真实 IP 获取）
  - 限流触发后的用户体验（429 响应 vs 静默降级）
- [ ] 每个盲区配一个具体替代/建议
- [ ] **不**罗列风格类小事（如"建议加注释"、"考虑命名"）

## 反例（出现即 fail）
- Phase 2 输出空泛（"注意安全"、"考虑性能"）
- 把 5 个以上盲区一股脑列出
- 把业务决策（"要不要限流"）当盲区反驳
