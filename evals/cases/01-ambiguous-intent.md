# 用例：模糊意图触发 A/B 选项
**测试维度**：Phase 1 意图透视

## 输入
```
/prism 帮我把这个项目的文档整理一下
```

## 期望行为
- [ ] 输出表层 / 暗层 / 深层三层透视
- [ ] 识别"整理文档"的歧义（重排现有 vs 补全缺失 vs 重写风格 vs 验证内容真实性）
- [ ] 给出 A/B（或更多）选项，每项一句话说明
- [ ] **不替伙伴下结论**，明确等待选择

## 反例（出现即 fail）
- 直接开始扫文件、列出"我打算这么整理"
- 只输出一段笼统的"好的我来帮您整理"
- 在没有澄清前就写入 active-plan.md
