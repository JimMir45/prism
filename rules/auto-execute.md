# 自主执行规则

## Compact 后恢复

当对话经历 compact（自动或手动）后，**立即检查**当前项目是否存在 `.claude/active-plan.md`。

如果存在且有未完成步骤（未勾选的 `- [ ]`）：
1. 读取该文件
2. **校验 worktree**：如计划含 `worktree:` 字段且与当前工作目录不一致，输出"`此计划属于 {worktree}，当前位于 {pwd}，可能是另一个 worktree 的进行中任务，是否切换 / 在此继续 / 忽略？`"等伙伴决策；字段缺失则视作匹配通过（向后兼容）
3. 输出一行恢复提示："`检测到未完成计划：{任务一句话}，创建于 {created_at}，{N}/{总数} 步已完成，即将从步骤 {下一步编号} 继续`"
4. **等待伙伴确认后才继续执行**（输入任意内容即视为确认）
5. 超过 24 小时未更新的计划文件，提示伙伴："此计划已超过 24 小时未更新，建议重新确认整个计划"

如果不存在或所有步骤已完成：正常交互，不做特殊处理。

如果伙伴的第一条消息是一个明确的新任务指令（与旧计划无关）：先处理新任务，之后提示"存在未完成的旧计划，是否继续/放弃？"

## 执行期间行为

当 `.claude/active-plan.md` 存在且有未完成步骤时：
- **不要在每一步完成后停下来汇报等待确认**
- 每完成一步，更新计划文件（勾选 + 追加执行日志），然后直接进入下一步
- 仅在以下情况暂停：
  - 遇到计划中标记的决策点
  - 遇到无法自行解决的错误（重试 1 次仍失败；重试前确认操作是否幂等，非幂等操作首次失败即暂停，不重试）
  - 遇到下方"不可逆操作清单"中的操作

## 不可逆操作清单（无论是否标记为决策点，一律暂停等确认）

- `git push`（含 force push）
- `rm -rf` 或批量删除文件/目录
- 数据库 `DROP` / `TRUNCATE` / 批量 `DELETE`
- 外部 API 的 `POST` / `PUT` / `DELETE` 调用
- 修改 `.env` 或包含凭据/密钥的文件
- 安装或卸载系统级依赖
- 修改 CI/CD 配置文件
- 部署命令（deploy、publish）

## 计划文件规范

- 路径：`{项目根}/.claude/active-plan.md`（项目级，多项目并行互不干扰）
- 由 `/prism` skill 的 Phase 4 创建
- 执行完毕后删除（git commit 本身就是归档）
- 同一项目同一时间只允许一个 active-plan
- `.claude/` 目录应加入 `.gitignore`
- `worktree:` 字段记录创建时的绝对路径，多 worktree 并发时用于校验
